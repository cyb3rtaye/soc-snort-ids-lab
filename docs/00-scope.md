# Scope & Success Criteria

## Objective
Deploy Snort on Ubuntu as an IDS sensor and produce evidence of :
- visibility into network traffic
- detection of baseline activity
- detection of controlled suspicious behaviour
- repeatable triage workflow

## Success Criteria
- [ ] Snort installed and validated (`snort -T`)
- [ ] HOME_NET configured correctly
- [ ] At least 2 custom rules documented and tested
- [ ] Alerts captured with timestamps and context
- [ ] Write-up with evidence and recommendations
